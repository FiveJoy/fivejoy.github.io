1. 小和问题 SmallSum

Location: https://github.com/FiveJoy/Questions/tree/master/src/com/fivejoy/smallsum

给出一个数组，对该数组的每一个元素M，求其左侧所有比M小的元素的和。将和累加并返回。
① 使用暴力遍历作为对数器
② 改归并排序
对该数组的每一个元素M，求其左侧所有比M小的元素的和；
其实也可以反过来看：对数组中每一个元素M，看它右侧有多少个元素比它大(假设有K个)，就有a*k个小和。最后累加即可
【对每个数求其右侧有多少个数比它大】
///最后的小和=左侧的小和+右侧的小和+本次merge产生的小和

2. 奇偶分区

实际是（快排如何实现为稳定的）&& 额外空间复杂度 O(1)
给定个数组，将数组中奇数放在左侧，偶数放在右侧-并且保证奇数和偶数相对次序都不变
是论文级别的- 01stable sort【快排的稳定实现法】有空可了解一下–暂时不必实现
归并-归并排序内部缓存法实现O(1)的额外空间

4. 最大差值MaxDiff

数组无序–数组排序后相邻两个元素的最大差值是多少？O(N)—问清元素是什么类型-long long long 的类型
解：O(N)所以普通排序肯定是不行，long long long 的类型 无法确定数据范围-无法使用计数排序

N个数 N+1个桶
不必关心一个桶内部的相邻两数的最大差值。
差值最大 可能出现 ①相邻两个非空桶(eg.10,29在两个不同的桶&差值为19) ②中间有个空桶的两个非空桶(eg.19,30在两个不同的桶&差值为11)

只是借用了桶的概念，却没有实际进行排序，只是收集各个桶的信息
【桶排序无非是Bucket中让人最容易理解的概念】

9 哈希表&哈希函数

hashcode()怎么实现：通过md5或者sha1
hashcode的性质：
① 输入域无限大；输出域有限大
② same input，same output
③ different input->may be same output
④ 离散性-
哈希表就是散列表，链表会均匀增长-

HashMap常用函数
map.containsKey(“key”);
map.get(“key”);
map.remove(“key”);
map.keySet();
map.values();–String value : map.values();
map.entrySet();
HashMap原来是数组+链表，后为提升查找效率改为数组+红黑树

实现RandomPool，要求实现GetRandom()平均概率取得元素
①设置两个HashMap KeyIndexMap&IndexKeyMap
②在删除的时候将 要删除的元素的index与lastindex的key换一下。保证0-index之间的元素都存在-好方便GetRandom的时候使用Math.random()
③本以为一个map也可以做到,只有KeyIndexMap但是要知道你是无法根据value也就是index的值getKey的，HashMap只有value=map.get(key);
④只有IndexKeyMap也不行。在delete的时候是通过key找到那个entry的。如果只有IndexKeyMap难道要遍历找到值么？那就太麻烦了。

12.在行和列都排好序的数组中判断是否存在元素K

【要求】时间复杂度为O(N+M)，额外空间复杂度为O(1)
自己想的时间复杂度并没有真的减小还是N^2，只不过最后的真实值变小了而已
【正解】初始点 -——右上角的点。如果该点比K大，向下移动，如果比K小向左移动;
同理使用左下角也可以。但是不可以是左上角和右下角—-不会淘汰任何排元素
